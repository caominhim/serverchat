Server
to Run: execute start.bat within the Server folder
type help for a list of commands

gen - 	generate a user id and key and add them to the list of saved keys.
	This also generates a file that the Client will need to place in their
	jar's directory

display - display all currently accepted keys and IDs

exit -	close the server and save keys (not calling exit will not save the 
	current key set)
current - display currently logged in IDs

Client
to Run: execute go.bat in any of the client folders

Without an Info.txt the client will notify that the file is missing and then
not work as expected

Info.txt should be the file generated by the server, renamed to Info.txt
